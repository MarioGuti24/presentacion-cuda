01 — Introducción

¿Qué es CUDA?

Plataforma de computación paralela desarrollada por NVIDIA que permite usar las GPU para cálculo científico de propósito general.

"La CPU organiza y coordina; la GPU resuelve en paralelo la parte intensiva del cálculo."
🚀

Lanzamiento

Introducida en 2006 bajo el liderazgo de Ian Buck. Supera las limitaciones de la CPU en tareas altamente paralelizables.

🔧

Herramientas

Incluye el compilador nvcc, librerías optimizadas y un entorno completo para desarrollar programas paralelos.

🧬

Aplicaciones

Inteligencia Artificial, simulaciones científicas, análisis de datos masivos, procesamiento de imágenes, física computacional.

El problema que resuelve

El crecimiento exponencial de datos y la complejidad de los problemas computacionales ha impulsado la necesidad de nuevas formas de procesamiento. En lugar de ejecutar tareas de forma secuencial, la computación paralela permite dividirlas y ejecutarlas simultáneamente, reduciendo drásticamente los tiempos de cómputo.

02 — Fundamentos

CPU vs GPU

Dos arquitecturas distintas y complementarias. La CPU optimiza la latencia; la GPU maximiza el rendimiento paralelo (throughput).

🧠

CPU

El "cerebro" del sistema. Pocos núcleos altamente potentes, diseñados para ejecutar tareas variadas con baja latencia.

  • NúcleosPocos — muy potentes
  • TareasSecuenciales y complejas
  • ObjetivoMinimizar latencia
  • Eficiencia paralelaLimitada
  • Uso típicoSO, apps generales

De los videojuegos al cálculo científico

Las GPU surgieron para acelerar el renderizado gráfico. Con el tiempo, incorporaron miles de núcleos capaces de operaciones simultáneas, dando lugar al concepto de GPGPU — computación de propósito general en GPU. CUDA fue la plataforma que democratizó este acceso para el cálculo científico.

03 — Hardware

Arquitectura Interna

La GPU se organiza en Streaming Multiprocessors (SM), cada uno con cientos de CUDA Cores trabajando de forma coordinada.

🏗️

Streaming Multiprocessors (SM)

Cada SM agrupa varios CUDA Cores, registros, memoria compartida y unidades de control. Son los bloques fundamentales de la GPU. Una GPU moderna puede tener decenas de SMs.

⚙️

CUDA Cores

Núcleos simples y especializados en operaciones matemáticas. No equivalen a núcleos CPU — son más simples pero se integran en cantidades masivas, ejecutando miles de operaciones simultáneas.

04 — Programación

Modelo de Programación

CUDA se basa en tres conceptos clave: la arquitectura Host/Device, la jerarquía de hilos y el Kernel.

Host & Device

CPU + GPU trabajando juntas

En CUDA existe una arquitectura heterogénea donde ambos procesadores colaboran. La CPU prepara, organiza y lanza; la GPU ejecuta en paralelo.

Jerarquía de Hilos

Grid → Blocks → Threads

Grid

Grid

Conjunto total de bloques que participan en la ejecución de un kernel. Representa todo el problema lanzado a la GPU.

Block

Block (Bloque)

Grupo de hilos que trabajan juntos, pueden compartir recursos y sincronizarse entre sí. Organiza el cálculo en partes manejables.

Thread

Thread (Hilo)

Unidad básica de ejecución. Cada hilo resuelve una pequeña parte del problema. Por ejemplo, calcular un único elemento de una matriz.

Visualización
El Kernel

La función que se paraleliza en GPU

El kernel contiene la operación elemental que CUDA repite masivamente. Se escribe una sola vez pero lo ejecutan miles de hilos al mismo tiempo, cada uno sobre datos diferentes.

// Suma de vectores — cada hilo calcula un elemento
__global__ void vecAdd(float* A, float* B, float* C) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    C[i] = A[i] + B[i];
}

// Multiplicación de matrices — cada hilo calcula C[row][col]
__global__ void matMul(float* A, float* B, float* C, int N) {
    int row = blockIdx.y * blockDim.y + threadIdx.y;
    int col = blockIdx.x * blockDim.x + threadIdx.x;
    float sum = 0.0f;
    for (int k = 0; k < N; k++)
        sum += A[row * N + k] * B[k * N + col];
    C[row * N + col] = sum;
}
🔁

Descomposición de datos

Cada hilo resuelve una parte independiente del problema — ideal para operaciones sobre vectores, matrices e imágenes.

📐

Granularidad fina

CUDA trabaja con miles de tareas pequeñas organizadas jerárquicamente para maximizar el uso de los CUDA Cores.

🔄

Modelo SPMD

El mismo kernel se ejecuta muchas veces sobre datos distintos — Single Program Multiple Data — principio fundamental de la computación paralela.

05 — Caso Práctico

Multiplicación de Matrices

A × B = C — problema clásico del cálculo numérico, con implementaciones radicalmente distintas en CPU y GPU.

🐢

Algoritmo CPU (Secuencial)

Tres bucles anidados que calculan cada elemento de C uno tras otro. Simple pero lento.

for(i = 0; i < N; i++)
  for(j = 0; j < N; j++)
    for(k = 0; k < N; k++)
      C[i][j] += A[i][k] * B[k][j];
  • ComplejidadO(n³)
  • ParalelismoNinguno
  • EscalabilidadMuy limitada
🚀

Algoritmo GPU (CUDA)

1 hilo = 1 elemento de C. Todos los elementos se calculan simultáneamente.

// Cada hilo calcula C[row][col]
int row = blockIdx.y * blockDim.y
        + threadIdx.y;
int col = blockIdx.x * blockDim.x
        + threadIdx.x;
for(k) sum += A[row][k] * B[k][col];
C[row][col] = sum;
  • ComplejidadO(n²) efectivo
  • ParalelismoMasivo
  • EscalabilidadExcelente
🔄

07 — Ecosistema

Librerías CUDA

No siempre hace falta implementar todo desde cero. NVIDIA proporciona librerías altamente optimizadas para los casos de uso más comunes.

cuBLAS
Álgebra lineal — multiplicaciones de matrices, vectores. Constante en cálculo científico y machine learning.
cuDNN
Redes neuronales profundas. Base de frameworks como TensorFlow y PyTorch para entrenamiento de IA.
Thrust
Paralelismo con una API similar a C++ STL. Facilita el trabajo sin bajar al nivel de los kernels.
cuFFT
Transformada Rápida de Fourier en GPU. Fundamental en procesamiento de señales y simulaciones.
cuSPARSE
Álgebra lineal para matrices dispersas. Útil en simulaciones físicas y grafos.
nvcc
Compilador oficial CUDA. Convierte código C++/CUDA en instrucciones ejecutables por la GPU.
